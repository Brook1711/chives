# 网站的后端

根据网站的查找和发送弹幕的功能，后端的逻辑基本可以分为收发两个模块，需要和前端统一一下接口API的参数：

## 后端具体函数逻辑

1. 模块一，收：
  - 功能：由前端传递字符串，后端在数据库中查找弹幕并返回
  - 伪代码：
    ```go
    fun(str 前端传递的电影名称):
        return 返回Json格式的弹幕
    ```
2. 模块二，发：
  - 功能：将前端传递来的弹幕插入到数据库中
  - 伪代码
    ```go
    fun(json 前端传递的json格式的弹幕):
        return 是否成功插入的消息
    ```
3. 补充：
  - 未来在收模块中查找弹幕方面会应用hash匹配等技术实现精确匹配，这里就先采用字符串匹配做demo
